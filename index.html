Learning Path

Sharks have to constently swim to stay alive, otherwise water won't pump through their gills to breathe. Similarly, software engineers have to contantly learn everyday and develop new skills otherwise they won't be able to keep up with new software technology.

I want to start off a new career as a frontend web developer. To achieve this I need to:

  1. Have HTML and CSS become second nature.
  2. Learn and become comfortable with Javascript.
  3. Use and keep on learning Python.
  4. Get familiar with Data Structures and Algorithms.
  5. Practice daily, make coding a nice routine (e.g Codewars).
  6. Final, Interview prep.

Road Blocks

Learning code is not an easy thing to do, theirs a reason why developers are paid what they're paid.

Some of the road blocks when becoming a software engineer are:

  • Not having enough patience.
  • Losing motivation after seeing everything you don't know.
  • Losing interest/not going through with it.
  • Stuck in tutorial hell, where you watch tutorial after tutorial, procastinating and not doing it by yourself.
  • Losing focus.
  • Being scared of going with it and failing.

Strategies

Sometimes it's not always that simple so these are some of my personal strategies used to steer me into becoming a web developer.

  • Getting through all of FreeCodeCamp.
  • Studing data structures and algorithms daily. (Perferrably after learning Javascript)
  • Putting a hold on playing video games and watching shows when I could be coding.
  • Putting a limit on going out as much, which would also save me money.
